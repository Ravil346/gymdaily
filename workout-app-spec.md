## Что это

PWA (Progressive Web App) для ведения дневника силовых тренировок. Запускается на iPhone 16e через Safari, устанавливается на главный экран, **работает полностью локально без интернета** после установки. Без бэкенда, без облака — все данные хранятся в IndexedDB на устройстве.

Цель — личный дневник для одного пользователя (меня). Production-уровень не нужен, важна простота и работоспособность.

## Модель данных

### Dexie schema

```typescript
import Dexie, { Table } from 'dexie';

export interface Program {
  id?: number;
  name: string;
  isFavorite: boolean;
  createdAt: number;
}

export interface ProgramDay {
  id?: number;
  programId: number;
  name: string;       // "День A", "Верх", и т.д.
  order: number;      // порядок в программе
}

export interface ProgramExercise {
  id?: number;
  dayId: number;
  name: string;
  sets: number;       // целевое число подходов
  reps: number;       // целевые повторы (целое; дробные допускаются только в записи)
  weight: number;     // стартовый вес, кг (≥ 0, шаг 0.25)
  progressionStep: number; // шаг прибавки кг (0 = без автопрогрессии)
  comment: string;
  order: number;
}

export interface Workout {
  id?: number;
  date: number;             // timestamp начала тренировки
  programId: number;
  dayId: number;
  finishedAt?: number;      // timestamp завершения
  note?: string;
}

export interface WorkoutExercise {
  id?: number;
  workoutId: number;
  programExerciseId: number; // ссылка на шаблон в программе
  name: string;              // копия из шаблона на момент старта (или переименованное)
  order: number;
  comment: string;
}

export interface WorkoutSet {
  id?: number;
  workoutExerciseId: number;
  setNumber: number;
  weight: number;     // ≥ 0, дробное допускается
  reps: number;       // ≥ 0, дробное допускается
  completedAt: number;
}

export class WorkoutDB extends Dexie {
  programs!: Table<Program>;
  programDays!: Table<ProgramDay>;
  programExercises!: Table<ProgramExercise>;
  workouts!: Table<Workout>;
  workoutExercises!: Table<WorkoutExercise>;
  workoutSets!: Table<WorkoutSet>;

  constructor() {
    super('WorkoutDB');
    this.version(1).stores({
      programs: '++id, isFavorite',
      programDays: '++id, programId, order',
      programExercises: '++id, dayId, order',
      workouts: '++id, date, programId, dayId',
      workoutExercises: '++id, workoutId, programExerciseId',
      workoutSets: '++id, workoutExerciseId',
    });
  }
}

export const db = new WorkoutDB();
```

### Ключевая логика

**При старте тренировки:**
1. Создаётся запись `Workout` (programId, dayId, date)
2. Для каждого `ProgramExercise` дня создаётся `WorkoutExercise` (копируется `name`, `comment`)
3. Подходы записываются по мере выполнения через `WorkoutSet`

**Замена упражнения** — редактируется `WorkoutExercise.name` для конкретной тренировки. `ProgramExercise` не трогается.

**"Прошлый раз" этого упражнения** — последний `WorkoutExercise` (по `workout.date`), у которого совпадает `programExerciseId`. Показываем его реальное `name` (если в прошлый раз были брусья — показываем брусья), и его подходы.

## Логика "следующего дня"

```typescript
function getNextDay(programId: number): ProgramDay {
  const days = await db.programDays
    .where('programId').equals(programId)
    .sortBy('order');

  const lastWorkout = await db.workouts
    .where('programId').equals(programId)
    .reverse()
    .first();

  if (!lastWorkout) return days[0]; // первая тренировка → первый день

  const lastDayIndex = days.findIndex(d => d.id === lastWorkout.dayId);
  const nextIndex = (lastDayIndex + 1) % days.length;
  return days[nextIndex];
}
```

Пользователь может вручную выбрать другой день перед стартом тренировки (чипы A/B на главной).

## Логика автопрогрессии

Перед стартом тренировки для каждого упражнения дня:

```typescript
function getRecommendedWeight(programExerciseId: number): {
  weight: number;
  willProgress: boolean;
} {
  const tpl = await db.programExercises.get(programExerciseId);

  // последний WorkoutExercise по этому шаблону
  const lastWE = await db.workoutExercises
    .where('programExerciseId').equals(programExerciseId)
    .reverse()
    .first();

  if (!lastWE) return { weight: tpl.weight, willProgress: false };

  const sets = await db.workoutSets
    .where('workoutExerciseId').equals(lastWE.id!)
    .toArray();

  // нужно: ровно `tpl.sets` подходов, в каждом reps >= tpl.reps
  const allDone =
    sets.length >= tpl.sets &&
    sets.slice(0, tpl.sets).every(s => s.reps >= tpl.reps);

  const lastWeight = sets[0]?.weight ?? tpl.weight;

  if (allDone && tpl.progressionStep > 0) {
    return {
      weight: lastWeight + tpl.progressionStep,
      willProgress: true
    };
  }
  return { weight: lastWeight, willProgress: false };
}
```

На главной у упражнений с `willProgress: true` показывается зелёная подсказка `↑ +X кг`.

### 2. Календарь — особое скролл-поведение

Это важный нюанс, прошу реализовать аккуратно.

**Идея:** наверху экрана — сетка месяца, ниже — список тренировок месяца. При скролле должны "сцепляться":

- **Изначально:** виден полный месяц + начало списка (первые 1–2 карточки)
- **Скролл вниз** (читаешь список): сначала календарь скроллится вверх вместе с контентом и **пропадает**. Дальше скроллится только список.
- **Скролл вверх** обратно: сначала растёт список вверх, потом снова появляется календарь, возвращаясь в исходное положение.

То есть это **не sticky-календарь** (который бы оставался наверху). Это обычный документ со скроллом, где календарь — часть документа сверху, и при скролле он естественно уезжает вверх, а при возврате — возвращается. Реализуется простым `overflow-y: auto` на контейнере.

**Детали:**
- Заголовок месяца + стрелки навигации ‹ › наверху
- Сетка 7×N с круглыми днями
- Сегодняшний день — обведён лавандовой рамкой (без заливки), если без тренировки
- Дни с тренировкой — лавандовая заливка + буква дня программы (A/B) мелко снизу
- Сегодня + тренировка — лавандовая заливка + рамка
- Между календарём и списком — отступ 28px
- Заголовок "N тренировок в [месяце]" перед списком
- Карточки тренировок: дата (день недели сокращённо), тег "День A", сводка "4 упражнения · 18 подходов" (без объёма в кг)
- Тап на карточку → деталь тренировки

### 6. Активная тренировка — особое внимание

Полноэкранный режим, **без нижнего таб-бара**, чтобы максимум места.

**Шапка:**
- Слева — кнопка ✕ (выйти, с подтверждением "Прервать тренировку?")
- По центру — прогресс "2 из 4" (текущее упражнение / всего)
- Справа — "Завершить" (доступна на любом упражнении; подтверждение если ещё не все упражнения сделаны)

**Контент — горизонтальный свайп:**
- Слева (основная страница): экран ввода подходов
- Справа (свайп вправо открывает): история этого упражнения

Реализовать через `scroll-snap-type: x mandatory` на горизонтальном scroll-контейнере или через CSS Carousel API / framer-motion.

**Основная страница (ввод подходов):**

1. Метка "Упражнение N из M" (мелко, лавандой)
2. Название упражнения (28px bold) — тапаемое; есть рядом маленькая кнопка-чип "✎ Изменить название" (для разовой замены)
3. Целевая инфо мелким текстом: "Цель: 5 × 5 · 60 кг" (на лёгкой плашке `var(--surface-2)`)
4. Список подходов, по одному на строку:
   - **Сделанные**: вес × повторы (полная заливка surface)
   - **Текущий**: подсветка `var(--accent-bg)`, надпись "Текущий"
   - **Будущие**: пунктирная рамка, прочерки
5. Поля ввода (только для текущего подхода):
   - Два инпута бок-о-бок: "Вес, кг" и "Повторы" (крупно, по центру)
   - Кнопка "Записать подход" (primary CTA)
6. Снизу мелко: подсказка "← свайп вправо · история этого упражнения"

**После записи последнего запланированного подхода:**
- Появляется кнопка "Следующее упражнение" → переход к следующему
- На последнем упражнении: "Завершить тренировку"
- Можно добавить дополнительный подход сверх плана (кнопка "+ Ещё подход")

**Изменение названия:**
- Тап на чип "Изменить название" → инлайн-инпут или модалка
- Меняется только `WorkoutExercise.name` (для этой тренировки)
- В программе ничего не меняется

**Страница истории (свайп вправо):**
- Метка "Прошлый раз"
- Название упражнения (как было в прошлый раз — может отличаться: брусья вместо жима)
- Дата ("3 мая · 7 дней назад")
- Таблица подходов: # / Вес / Повторы
- Если упражнения раньше не было: вместо таблицы — текст "Раньше не выполнялось"

### 7. Деталь прошедшей тренировки

Открывается из календаря.
- Кнопка "‹ Календарь" слева, "Изменить" справа
- Шапка: дата + день программы + название программы
- Для каждого упражнения:
  - Название (если переименовано — в скобках исходное)
  - Таблица подходов (# / Вес / Повторы)
- Опционально — комментарий к тренировке

В режиме "Изменить" можно править веса/повторы (на случай ошибки записи), удалить тренировку.

## Поведение чисел

### Ввод

- Вес (`weight`): число ≥ 0, шаг 0.25, **может быть нулевым** (0 = без отягощения, например подтягивания с массой тела) и **дробным** (60.5)
- Повторы (`reps`): число ≥ 0, **может быть нулевым** (0 = подход не сделан / провал) и **дробным** (5.5, 3.5)

В инпутах принимай и точку, и запятую как разделитель.

### Отображение

Использовать русскую локаль через `toLocaleString('ru-RU')`:
- `60` → "60 кг"
- `60.5` → "60,5 кг"
- `5.5` → "5,5 повт"

Для целых не показывать `,0`.

### Service Worker

Через `vite-plugin-pwa`:
- `registerType: 'autoUpdate'`
- `workbox: { globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'] }`
- Все ассеты должны кешироваться на установке, чтобы дальше работать офлайн

### iOS-специфика

В `index.html` добавить:

```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="apple-mobile-web-app-title" content="Тренировки">
<link rel="apple-touch-icon" href="/icons/apple-touch-icon.png">
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover">
```

`viewport-fit=cover` нужен для корректной работы safe-area-inset.

### Safe area

Везде, где нужно отступать от краёв экрана с вырезом или с индикатором Home:

```css
.app-container {
  padding-top: env(safe-area-inset-top);
}

.bottom-nav {
  bottom: max(16px, env(safe-area-inset-bottom));
}

.scrollable-content {
  padding-bottom: calc(80px + env(safe-area-inset-bottom));
}
```

## Замечания о макете

К этому ТЗ прилагается файл **`workout-mockup-v2.html`** — визуальный референс. Пожалуйста, учти:

1. **Это макет, не точный гайд.** В нём могут быть мелкие дефекты — где-то контент налезает на нижнее меню, где-то отступы неточные. В реальной реализации этого быть не должно: контент должен корректно отступать от капсулы (см. секцию про padding-bottom).

2. **Берём из макета:** палитру (CSS-переменные точно), шрифт Quicksand, форму элементов (радиусы, тени), общую структуру каждого экрана.

3. **Не копируем буквально:** конкретные пиксельные значения отступов нужно подбирать под реальные размеры iPhone 16e и удобство использования. Если что-то в макете выглядит криво — делай правильно, не повторяй ошибку.

4. **Иконки в макете** — частично эмодзи и заглушки. В коде используй нормальные SVG (lucide-react или аналог).

5. **Скролл-поведение календаря** в макете не реализовано — там просто статичная страница. Реализуй как описано в секции "Календарь".
